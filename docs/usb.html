Prima di iniziare / Before you start

  1. Scarica l'ISO dalle Releases (es. quelo-doctor-0.75-alpha.iso, ~1 GB).
  2. Usa una chiavetta da almeno 2 GB (consigliato 4 GB+). Tutti i dati sulla USB verranno cancellati (tranne con Ventoy se aggiungi solo un file ISO).
  3. Identifica il dispositivo giusto — confondere disco interno e USB è il modo più comune per perdere dati.
  4. Dopo la scrittura, riavvia il PC e avvia da USB (vedi Avvio PC).

Come trovare il nome del dispositivo USB

SistemaComando / strumento
Linuxlsblk -o NAME,SIZE,MODEL,TRAN — la USB è di solito /dev/sdX (es. sdb), non sdb1
macOSdiskutil list — dispositivo /dev/diskN
WindowsGestione disco o l'interfaccia del programma (Rufus, Etcher…) — lettera o disco fisico

Attenzione: con dd e strumenti “raw” scrivi sull'intero disco (/dev/sdb), non sulla partizione (/dev/sdb1).

1. Ventoy consigliato per tecnici

Ventoy formatta la USB una sola volta; poi copi le ISO come file (trascinamento o copia normale). Puoi tenere più ISO sulla stessa chiavetta e scegliere al boot.

Pro: niente riscrittura completa a ogni aggiornamento · più ISO sulla stessa USB · UEFI + BIOS
Contro: primo setup leggermente più lungo

  1. Scarica Ventoy da ventoy.net (Windows, Linux o macOS).
  2. Installa Ventoy sulla chiavetta con Ventoy2Disk (GUI) o Ventoy2Disk.sh -i /dev/sdX (Linux).
  3. Al termine, la USB avrà una partizione dati (es. etichetta Ventoy).
  4. Copia quelo-doctor-….iso in quella partizione — come un file qualsiasi.
  5. Boot da USB → menu Ventoy → seleziona quelo-doctor-….iso → Invio.

Per aggiornare Quelo Doctor: sostituisci il file ISO sulla USB. Non serve rifare Ventoy.

2. dd (Linux / macOS)

Scrittura bit-per-bit dell'ISO sull'intera chiavetta. Metodo classico, veloce, senza GUI.

Linux

# 1) Trova la USB (es. sdb) — NON il disco del sistema
lsblk -o NAME,SIZE,MODEL,TRAN

# 2) Smonta eventuali partizioni
sudo umount /dev/sdb* 2>/dev/null

# 3) Scrivi l'ISO (sostituisci sdb e il nome file)
sudo dd if=quelo-doctor-0.75-alpha.iso of=/dev/sdb bs=4M status=progress conv=fsync

# 4) Sincronizza
sync

macOS

diskutil list
diskutil unmountDisk /dev/diskN
sudo dd if=quelo-doctor-0.75-alpha.iso of=/dev/rdiskN bs=4m status=progress
sync

Su macOS usa rdiskN (raw) per velocità maggiore. diskN è il numero del dispositivo USB.

3. balenaEtcher

balenaEtcher — GUI semplice per Windows, macOS e Linux. Adatto a chi preferisce non usare la riga di comando.

  1. Installa o avvia balenaEtcher.
  2. Flash from file → seleziona quelo-doctor-….iso.
  3. Select target → scegli la chiavetta USB (controlla capacità ed etichetta).
  4. Flash! — attendi fine e verifica eventuale.

Etcher scrive in modalità raw come dd: la USB conterrà solo questa ISO fino a una nuova scrittura.

4. Rufus (Windows)

Rufus è molto usato su Windows per ISO avviabili Linux.

  1. Scarica Rufus (portable o installer).
  2. Dispositivo → seleziona la USB.
  3. Selezione boot → Disco o immagine ISO → Scegli → file quelo-doctor-….iso.
  4. Schema partizione: di di solito GPT + UEFI su PC moderni; MBR + BIOS su hardware vecchio. In dubbio: GPT/UEFI (il PC legacy spesso ha comunque CSM).
  5. Avvia → conferma cancellazione dati sulla USB.

Per ISO Linux live, lascia Rufus in modalità ISO (default). Non serve “DD image” salvo problemi di boot.

5. GNOME Disks / KDE ISO Image Writer (Linux)

GNOME Dischi (Disks)

Presente su Ubuntu, Fedora Workstation e molte distro con GNOME.

  1. Apri Dischi / Disks.
  2. Seleziona la chiavetta USB nella colonna sinistra.
  3. Menu ⋮ → Restore Disk Image… / Ripristina immagine disco….
  4. Scegli l'ISO Quelo Doctor → conferma.

KDE ISO Image Writer

Su KDE Plasma: cerca ISO Image Writer nel menu applicazioni, seleziona ISO e destinazione USB.

6. USBImager

USBImager — programma leggero, open source, per Windows, Linux e macOS. Interfaccia minimale simile a Etcher.

  1. Avvia USBImager.
  2. Select image → ISO Quelo Doctor.
  3. Select disk → USB corretta.
  4. Write → attendi completamento.

7. Win32 Disk Imager (Windows)

Win32 Disk Imager — storico su Windows, ancora usato per scritture raw.

  1. Avvia come amministratore se richiesto.
  2. Image File → icona cartella → seleziona .iso.
  3. Device → lettera della USB (verifica in Esplora file).
  4. Write → conferma.

Controlla due volte la lettera unità: Win32 Disk Imager non ha le stesse protezioni di Etcher.

8. Altri metodi validi

ToolPiattaformaNote
Fedora Media Writer Windows, macOS, Linux Pensato per Fedora ma accetta anche altre ISO; interfaccia guidata
Arch dd / cp Linux Stesso principio di dd; su alcuni sistemi cp iso /dev/sdX funziona ma dd è più portabile
pv + dd Linux pv quelo-doctor.iso | sudo dd of=/dev/sdX bs=4M — barra di avanzamento
UNetbootin Windows, Linux Meno usato oggi; verifica che l'ISO sia supportata come “disco live”
VM + pass-through USB Qualsiasi Per test in QEMU/VirtualBox senza scrivere USB fisica: monta l'ISO come CD avviabile

In generale: se il tool scrive l'ISO come immagine disco avviabile (non solo copia file dentro una partizione FAT), va bene. In dubbio, usa Ventoy, dd o balenaEtcher.

Avvio del PC / Booting the PC

  1. Inserisci la USB nel PC da riparare.
  2. Accendi e apri il menu boot o il setup firmware:
    • Tasti comuni: F12, F11, F9, F8, Esc, F2 (setup)
    • Il messaggio all'avvio indica il tasto (es. “Press F12 for boot menu”)
  3. Scegli la USB (può comparire come nome marca, “UEFI: …” o “USB HDD”).
  4. Se non parte: in UEFI disabilita temporaneamente Secure Boot o prova la voce Legacy / CSM.
  5. Dovresti vedere il logo Quelo Doctor e il menu testuale.

Requisiti hardware: vedi USB e requisiti minimi sulla pagina principale.

Problemi? / Troubleshooting

  • USB non compare nel menu boot — prova un'altra porta (USB 2.0 posteriore), disabilita Secure Boot, o riscrivi con un altro metodo.
  • Schermo nero dopo boot — attendi 30–60 s; prova modalità Legacy se UEFI fallisce.
  • “Operating system not found” — ISO scritta male o dispositivo sbagliato; ripeti con dd o Etcher.
  • Ventoy non elenca l'ISO — controlla che il file sia nella partizione dati Ventoy e che l'estensione sia .iso.
